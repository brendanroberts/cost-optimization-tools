# marginsteward.com — Change Requirements

## Context

marginsteward.com is a standalone, unbranded cost optimization resource for
finance professionals. It currently includes a scenario analysis tool, a
payroll savings page, and a printable checklist. It is intentionally generic —
no service promotion, no practitioner branding beyond a contact email in the
FAQ.

The site has a flow where the scenario tool includes a "Continue to Onboarding
Planner" button. This leads to a form that collects user inputs and submits
them to an undisclosed endpoint. This flow needs to be replaced with a
self-serve output page that delivers utility without any form submission.

---

## Change 1: Remove the Onboarding Planner Form

### Current behavior
- Scenario tool has a "Continue to Onboarding Planner" button
- Button carries current scenario state (selected categories and dollar
  amounts) into a subsequent page
- That page presents a form collecting user information
- Form submits to a hidden endpoint (not visible to the user)

### Required behavior
- Remove the form and all submission logic entirely
- Remove any hidden endpoint, webhook, or form handler associated with this
  flow
- Retain the button on the scenario tool page but rename it (see Change 2)

---

## Change 2: Rename and Reframe the Button and Destination Page

### Button
- Rename "Continue to Onboarding Planner" to something self-serve and
  utility-focused
- Suggested label: **"Generate My Preparation Checklist"** or **"View My
  Review Summary"**
- Tone should imply the user is getting something for themselves, not
  initiating contact with a practitioner

### Destination page
- Retain the display of categories and dollar amounts carried over from the
  scenario tool state
- Reframe the page as a **self-serve preparation summary**, not an intake form
- Page should feel like a useful output the user generated for themselves

---

## Change 3: Destination Page Content

The destination page should display the following, populated from scenario
state where applicable:

### Section 1: Your Selected Categories
- List each category the user added in the scenario tool
- Show the monthly and/or annual dollar amount they entered for each
- No editorial commentary — just a clean summary of their inputs

### Section 2: What to Gather
- For each selected category, display a short list of materials typically
  needed for a vendor expense review in that category
- This is static, category-specific content — not personalized
- Tone: practical and instructional, not promotional
- Example for Telecommunications:
  - Current invoices for all voice, data, and mobile services (last 3 months)
  - Copies of active contracts or service agreements
  - Inventory of lines, devices, and services currently active
  - Contact information for your current vendor account representative

### Section 3: General Preparation Notes
- A short static block of guidance that applies regardless of categories
  selected
- Example content:
  - Gather materials in digital form where possible
  - Note any contracts with upcoming renewal or expiration dates
  - Flag any services that have changed in scope but not in price

### Section 4: Printable/Exportable Output
- Page should be cleanly printable via browser print (CSS print styles)
- Optionally: add an "Export to PDF" button consistent with the existing
  export functionality on the scenario tool page
- No branding, no contact information, no calls to action on the printed
  output

## Change 5: Remove or Soften Any Remaining Promotional Language

### Audit the following for promotional tone:
- Any language that implies a specific practitioner is offering services
- Any language that describes the site as a lead generation or intake tool
- The FAQ answer to "Who created this tool?" — retain authorship credit but
  remove any language that reads as a service offer

### Acceptable tone:
> "This tool was developed by a cost optimization practitioner as a standalone
> resource for finance professionals."

### Remove or revise:
- Any phrasing like "reach out to discuss your situation" or similar soft CTAs
  that imply a service relationship

---

## Out of Scope for This Pass

The following are noted for future development but are not part of this
change request:

- Individual category landing pages (e.g. /telecom, /credit-card-processing)
- Email capture or content subscription functionality
- About or practitioner profile page
- Expanded FAQ content
- YouTube-linked landing pages

---

## Notes for Implementation

- The scenario tool state passed to the destination page should continue to
  work as-is — only the destination page content and structure changes
- All changes should preserve the existing site tone: clean, minimal,
  finance-professional
- No new external dependencies or tracking should be introduced
- Confirm that no form submission endpoints, webhooks, or third-party
  integrations remain active after the onboarding planner is removed
